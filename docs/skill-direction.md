# 스킬 방향

## 목적

- 비자명한 코드 변경을 목표에 비례하고, 근거가 있으며, 검증 가능한 상태로 유지한다.

## 유지할 것

- “필요한 일을 필요한 만큼 한다”를 중심 원칙으로 둔다.
- 직접 후속 수정은 누락하지 않는다.
- 주요 변경은 요청, 성공 기준, 기존 코드 제약 중 하나로 설명 가능해야 한다.
- 검증은 변경의 성격과 위험에 맞춘다.
- 모호성이 큰 결정은 질문하고, 작은 로컬 모호성은 합리적인 가정을 밝히고 진행한다.

## 피할 것

- “작게”를 이유로 필요한 호출부, 타입, 테스트, 문서, 설정 수정을 빼는 방향.
- “철저하게”를 이유로 무관한 정리, 넓은 리팩터링, 미래 대비 옵션을 섞는 방향.
- 사용자가 명시하지 않은 기능, 의존성, 설정, 추상화를 추가하는 방향.
- 검증 근거 없이 완료를 주장하는 방향.

## 수정 기준

- 반복해서 오해되는 판단 기준만 shipped skill에 추가한다.
- 사용자가 런타임에 참고해야 하는 내용은 repo-local docs가 아니라 `skills/justified-change/` 안에 둔다.
- 예시가 꼭 필요하면 `SKILL.md`에 아주 짧게 넣거나, 조건부로 읽을 수 있는 스킬 패키지 내부 reference로 분리한다.
- 한국어 문서는 영문 base skill의 의미를 보존하되 자연스럽고 직접적으로 쓴다.
- 저장소 운영 문서는 스킬 사용 계약을 대신하지 않는다.
